## Task
Generate test-case titles from the acceptance criteria below.

## Input
Acceptance criteria is provided as a LIST of BLOCKS.
Each block may contain zero or more lines starting with `Scenario:`.

Acceptance criteria blocks:
{acceptance_criterias}

## Output format (STRICT)
- Output MUST be a nested list (array of arrays).
- There MUST be exactly one inner list per acceptance-criteria block, in the SAME ORDER as the input.
- NEVER merge scenarios from different acceptance-criteria blocks into a single list.

## Rules
1. For each acceptance-criteria block:
   - Extract every line that starts with `Scenario:`
   - For each such line, create exactly one test-case title
2. Test-case title = only the text AFTER `Scenario:` (trim whitespace).
3. Do NOT add:
   - prefixes (e.g., Test, TC, ID)
   - descriptions, steps, Given/When/Then
   - invented or inferred scenarios
4. If an acceptance-criteria block contains NO `Scenario:` lines, return an empty list `[]` for that block.
5. Output plain text only.

## Output example (structure only)
[
  ["Scenario title 1 from block 1", "Scenario title 2 from block 1"],
  ["Scenario title 1 from block 2"],
  []
]

IMPORTANT:
- The number of inner lists MUST equal the number of acceptance-criteria blocks.
- Keep scenario titles exactly as written after `Scenario:`.
